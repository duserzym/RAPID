# RapidPy Up/Down Control

RapidPy Up/Down Control is the current Python operator panel for the RAPID Z axis. It keeps the VB6 raw-count motion model from `VB6/frmDCMotors.frm`, but consolidates motor, SQUID, vacuum, profile references, and measurement-Z optimization into one compact window.

## Current UI

- Header connections for the motor, SQUID, and vacuum serial lines
- Live status pills for Z raw position, Z in cm, MeasPos, target, and top-switch state
- Settings card for the active INI or JSON profile, MeasPos editing, and reference-height review
- Center Z-profile view with clickable labels and double-click motion targets
- `Velocity And Jog` card with VB6-compatible raw velocity, cm/s helper text, jog buttons, and raw target motion
- `Measurement Z Optimization` card with baseline capture, scan window setup, point-by-point scan logging, and fitted MeasPos suggestion
- Console that follows live output only while the operator is already scrolled to the end

## VB6 Transition Notes

- `Home To Top` tracks the legacy `HomeToTop` behavior from `frmDCMotors.frm`
- `Go Target`, `Move To Meas Target`, and jog actions follow the same raw-count motion model as `MoveMotor` and `UpDownMove`
- `Pickup`, `Dropoff`, and `Susc. Meter` preserve the older reference-position workflow while making the raw values editable in the Python UI
- Vacuum connect, reset, valve connect, and motor-power toggling follow the command flow from `VB6/frmVacuum.frm` and the magnetometer-side expectations in `VB6/frmMagnetometerControl.frm`
- The current Python safety model keeps the VB6 stop-and-warn outcome, but uses a MeasPos-relative software clip plus stopped-short detection because the shared Python motor wrapper does not yet expose a dedicated resistance/load bit

## Settings Workflow

- Default hardware/reference values load from `VB6/settings/Paleomag_v3.INI`
- The app can load either VB6-style `.ini` files or the newer section-based `.json` settings format
- Saving creates a snapshot under `.rapidpy_history/<settings-stem>/` before overwriting an existing settings file
- COM-port selections remain machine-local in `~/.rapidpy_updown_settings.json`

## Run

```powershell
conda activate paleomag
python RapidPy/updown_control/main.py
```

## Windows Build

```bat
cd RapidPy\updown_control
build_windows.bat
```

The build writes `dist\RapidPyUpDown.exe` in the repository root and now prefers the repository virtual environment automatically when `.venv\Scripts\python.exe` exists there.

## macOS Build

```bash
cd RapidPy/updown_control
chmod +x build_macos.sh
./build_macos.sh
```

## Notes

- Uses the shared hardware and UI layers from `RapidPy/rapidpy_common/`
- The profile view uses the same single-click select / double-click activate interaction style as the Python changer app
- Measurement-Z fitting now optimizes on the maximum absolute SQUID moment rather than only the most positive value
- Icon assets are generated by `tools/generate_icon.py` into `assets/updown_control_icon.{png,ico,icns}`
