# RapidPy DataViewer

RapidPy DataViewer is intended to stay lightweight. The goal is not to recreate the full `pmag_gui.py` or `demag_gui.py` workflow, but to give operators and students a fast live-view module they can open during measurement runs to inspect current data, check whether a directional component is emerging cleanly, and get a practical hint about the next demagnetization step.

## Current Scope

This module now supports:

- opening a CIT-format `.sam` file and browsing its linked specimen files
- opening a MagIC project directory that contains `measurements.txt`
- fallback loading of the older `label,n,e,u` CSV format for quick ad hoc inspection
- auto-detecting the experiment type for each specimen as `AF`, `Thermal`, `AF + thermal`, or `IZZI Thellier`
- a lightweight directional workspace with:
  - 2D component-style Zijderveld view
  - 3D vector view with a PCA fit line
  - equal-area stereonet
  - intensity decay curve
  - Arai-style paleointensity tab for IZZI / Thellier-style labels
- a quick next-step hint panel that suggests the next AF field, thermal step, or IZZI temperature pair and explains why
- optional scaling of point size by measurement CSD so noisier points are visually obvious
- automatic file watching with debounced reloads for `.sam`, specimen, CSV, or `measurements.txt` updates

## Lightweight Design Decision

The viewer should remain the "open it in the middle of a run" tool:

- no full interpretation editor
- no MagIC upload workflow
- no replacement for the heavier PmagPy GUIs
- fast specimen switching and quick read-only analysis first

That keeps this app useful at the instrument while still making it easier for students to answer questions like "what temperature should I do next?" without opening the larger PmagPy stack.

## Refined Plan

### Phase 1: Real data loading

Completed.

- Reuse local RAPID readers for CIT `.sam` and specimen files.
- Add a minimal MagIC `measurements.txt` reader for the columns needed by this viewer.
- Normalize all inputs into one common per-specimen model.

### Phase 2: Fast visualization surface

Completed in the lightweight sense.

- Add specimen browsing for multi-specimen CIT or MagIC datasets.
- Add a 2D component view that keeps the conventional open/closed projection feel.
- Add a lightweight 3D vector view with a PCA fit line and drag rotation.
- Add fit-range controls so the user can evaluate a chosen interval without launching the full interpretation GUI.

### Phase 3: Experiment-aware guidance

Completed as a first-pass rule-based system.

- Detect AF, thermal, AF+thermal, and IZZI-style sequences from step labels or MagIC treatment fields.
- Suggest a likely next step using remaining intensity, recent directional change, and experiment type.
- Show the reasoning next to the suggested step so the tool teaches, not just commands.

### Phase 4: Paleointensity quicklook

Partially completed.

- IZZI-style labels now produce an Arai-style quicklook tab and a simple linear trend summary.
- This is intentionally a rapid screening view, not a full paleointensity solution or alteration test workflow.

### Phase 5: Next increments

Still to do.

- Improve MagIC IZZI parsing when treatment metadata is richer than the measurement name.
- Add point picking directly inside the 2D and 3D views instead of only using the fit-range controls.
- Add more `demag_gui`-style interpretation summaries such as anchored / unanchored line options, great-circle quicklook, and additional fit diagnostics.
- Add better paleointensity checks such as pTRM check highlighting and alteration warnings.

## Machine Learning Note

Historic group data could become useful later, but it should not be the first solution.

Why not first:

- students need transparent reasoning, not a black-box temperature suggestion
- historic demagnetization decisions are usually sparse, inconsistent, and only weakly labeled unless someone curates them carefully
- rule-based advice tied to intensity decay, curvature, directional stability, and protocol type is easier to trust and debug

When ML may help:

- after there is a curated training set that records specimen type, protocol, each chosen next step, and whether the final interpretation was considered good
- as a second opinion layered on top of the rule-based hints, not as the only recommendation

In short: start with explainable heuristics, then evaluate whether a trained model adds value beyond them.

## Running

From the repo root:

```powershell
python RapidPy/data_viewer/main.py
```

Or from the module folder:

```powershell
python main.py
```

## Assets

- App icon assets are generated by `RapidPy/data_viewer/tools/generate_icon.py`.
- The current icon uses a pale goldenrod tile and a centered orange selection ring over one plotted data point.
